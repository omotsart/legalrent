# Пример трассировки

Одна запись из журнала трассировки (таблица `traces` в SQLite) — обезличенная.
Иллюстрирует аудируемость: по любому ответу видно, **как** система к нему пришла,
какое правило сработало и что именно попало в контекст. Все данные вымышлены.

Сценарий: пользователь спросил про пеню. Роутер по слабому триггеру сначала повёл
запрос в таблицы (`route_initial: sql`), там пусто → сработал fallback в документы
(`route_final: rag`). Оба маршрута зафиксированы — fallback виден, а не спрятан.

```json
{
  "trace_id": "a1b2c3d4e5f60718",
  "ts": "2026-07-21 10:42:15",
  "query": "какая пеня в договоре с ООО «Ромашка»?",
  "route_initial": "sql",
  "route_final": "rag",
  "rule_name": "d1_requisites",
  "matched_on": "пеня",
  "model": "GigaChat",
  "latency_ms": 1840,
  "n_chunks": 4,
  "answer_len": 212,
  "guard_fired": false,
  "playbook_version": null,
  "steps": [
    {"t": 0.002, "step": "route", "target": "sql", "rule": "v10_default", "matched": ""},
    {"t": 0.031, "step": "sql", "fallback": true, "empty": true},
    {"t": 0.033, "step": "sql_fallback_to_rag"},
    {"t": 0.402, "step": "retrieve", "n": 4},
    {"t": 0.418, "step": "context", "chars": 1876, "n_sources": 4},
    {"t": 1.836, "step": "generate", "model": "GigaChat"}
  ],
  "chunks": [
    {"source": "contract_042", "filename": "Договор аренды.docx", "clause_index": 41, "score": 0.83},
    {"source": "contract_042", "filename": "Договор аренды.docx", "clause_index": 40, "score": 0.71},
    {"source": "contract_042", "filename": "Договор аренды.docx", "clause_index": 52, "score": 0.55},
    {"source": "act_017",      "filename": "Акт приёма-передачи.docx", "clause_index": null, "score": 0.34}
  ]
}
```

## Как это читать

| Поле | Что показывает |
|---|---|
| `route_initial` → `route_final` | `sql` → `rag`: запрос ушёл в таблицы, там пусто, система сама перешла к документам. **Fallback — это история, а не молчание.** |
| `rule_name`, `matched_on` | какое правило приняло решение и на каком триггере (`«пеня»`) — маршрут объясним. |
| `steps[]` | пошаговый след с временными метками: видно, что 1,4 с из 1,84 с ушли на генерацию LLM, а не на поиск. |
| `chunks[].score` | релевантность каждого фрагмента; у акта `clause_index: null` — номер пункта не распознан, поле честно пустое. |
| `guard_fired` | детектор галлюцинаций не срабатывал: в ответе не было сумм/пунктов/дат вне источников. |
| `playbook_version` | `null` — запрос не про оценку правок; в трейсах redline здесь стоит версия playbook. |

Трассировка — это одновременно инструмент отладки, материал для метрик и основа
воспроизводимости для комплаенса: любой ответ можно восстановить по его следу.
